# Zeno MVP Specification

## MVP Scope: "Solo Creator"

The MVP focuses on a single user creating, iterating, and publishing dashboards. Team features, advanced access controls, and monetization come in later phases.

## What's IN the MVP

| Feature | Description | Priority |
|---------|-------------|----------|
| **Email Auth** | Magic link sign-in via Supabase | P0 |
| **Dashboard List** | View all my dashboards | P0 |
| **Create Dashboard** | New dashboard from scratch | P0 |
| **Data Input** | Paste CSV/JSON or upload file | P0 |
| **AI Generation** | Auto-generate initial charts from data | P0 |
| **Chat Iteration** | Modify dashboard via natural language | P0 |
| **Manual Editing** | Click to edit chart properties | P1 |
| **Publish Toggle** | Make dashboard publicly accessible | P0 |
| **Public Viewer** | Clean, read-only view for shared links | P0 |
| **Zeno Branding** | "Made with Zeno" badge on published dashboards | P0 |

## What's OUT of MVP

| Feature | Reason | Phase |
|---------|--------|-------|
| Team workspaces | Complexity | Phase 2 |
| Member invitations | Requires teams | Phase 2 |
| Dashboard sharing (editors/viewers) | Requires auth for viewers | Phase 2 |
| Access restrictions | Requires viewer auth | Phase 2 |
| Commenting | Requires viewer identity | Phase 2 |
| Data versioning UI | Nice-to-have | Phase 2 |
| Custom subdomains | Monetization feature | Phase 3 |
| Custom domains | Monetization feature | Phase 3 |
| Branding controls | Monetization feature | Phase 3 |
| Stripe billing | Monetization feature | Phase 3 |

## Data Models (MVP)

### User
```typescript
interface User {
  id: string;           // UUID
  email: string;        // Unique
  name?: string;
  created_at: Date;
}
```

### Dashboard
```typescript
interface Dashboard {
  id: string;           // UUID
  user_id: string;      // FK to User
  title: string;
  slug: string;         // Unique, for public URL
  data: object;         // The raw data (JSON)
  config: DashboardConfig;
  is_published: boolean;
  published_at?: Date;
  created_at: Date;
  updated_at: Date;
}
```

### Dashboard Config
```typescript
interface DashboardConfig {
  title: string;
  description?: string;
  charts: ChartConfig[];
  layout?: LayoutConfig;
  theme?: ThemeConfig;
}

interface ChartConfig {
  id: string;
  type: 'number_card' | 'line' | 'bar' | 'area' | 'pie' | 'table';
  title: string;
  config: {
    // Type-specific configuration
    // See AI Integration doc for details
  };
}
```

## MVP Acceptance Criteria

### Must Have (P0)
- [ ] User can sign up/login with email magic link
- [ ] User can see list of their dashboards
- [ ] User can create a new dashboard
- [ ] User can paste CSV data
- [ ] User can upload CSV/Excel file
- [ ] AI generates initial charts from data
- [ ] User can chat to modify dashboard
- [ ] User can publish dashboard
- [ ] Anyone with link can view published dashboard
- [ ] Published dashboard shows "Made with Zeno" badge

### Should Have (P1)
- [ ] User can manually edit chart titles
- [ ] User can manually change chart types
- [ ] User can manually adjust chart colors
- [ ] User can delete charts
- [ ] User can reorder charts
- [ ] User can unpublish dashboard
- [ ] User can delete dashboard

### Nice to Have (P2)
- [ ] User can duplicate dashboard
- [ ] User can rename dashboard
- [ ] Dashboard shows last updated time
- [ ] Loading states are smooth and informative
- [ ] Error states are helpful
